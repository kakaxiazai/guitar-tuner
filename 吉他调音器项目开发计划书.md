一、项目概述
1.1 项目背景
吉他调音器是吉他手日常练习和演出必备的工具。市面上的调音器应用功能繁杂，广告较多，且大多数将调音器和节拍器功能分开。本项目旨在开发一个简洁、专业、无广告的吉他调音器应用，集成自动调音、手动调音和节拍器三大核心功能，满足个人日常使用需求。
1.2 项目目标
- 开发一款跨平台的吉他调音器应用（优先支持iOS/Android）
- 实现精准的吉他调音功能，支持标准调弦（EADGBE）
- 提供自动和手动两种调音模式
- 集成专业节拍器功能，支持多种节拍类型和速率调节
- 界面简洁直观，操作便捷
- 响应速度快，音频处理延迟低于50ms
1.3 目标用户
- 吉他初学者：需要直观的调音指引和稳定的节拍参考
- 吉他进阶玩家：需要快速精准的调音和灵活的节拍器
- 个人日常使用：替代物理调音器和节拍器，减少设备携带
1.4 项目范围
本项目为个人使用的独立应用，不包含以下功能：
- 其他乐器的调音支持（如贝斯、尤克里里等）
- 在线功能（如云同步、社交分享等）
- 复杂的音频效果处理
- 多语言支持（优先中文和英文）
二、功能需求分析
2.1 吉他调音功能
2.1.1 自动调音模式
- 实时音频输入分析，自动识别当前弹奏的琴弦
- 显示音高偏差（以音分为单位，精度±1音分）
- 可视化调音指示器（指针式或LED式）
- 当音高准确时给出视觉和听觉反馈
- 支持标准调弦：E2(82.41Hz)、A2(110Hz)、D3(146.83Hz)、G3(196Hz)、B3(246.94Hz)、E4(329.63Hz)
- 噪声抑制和滤波处理，提高识别准确度
- 响应时间小于100ms
2.1.2 手动调音模式
- 用户手动选择需要调音的琴弦（6弦到1弦）
- 播放该琴弦的标准音高作为参考音
- 显示当前弹奏音高与标准音高的偏差
- 支持参考音的音量调节
- 支持参考音的持续播放或节拍播放
- 提供可视化波形显示，帮助用户判断音高
2.2 节拍器功能
2.2.1 速率控制
- 节拍范围：20 BPM - 280 BPM
- 调节精度：1 BPM
- 支持点击速度自动检测（Tap Tempo）
- 保存常用速度预设
2.2.2 节拍类型
- 基本节拍：2/4、3/4、4/4、5/4、6/8、7/8、9/8
- 每拍细分：支持1、2、3、4、6连音
- 重音设置：可自定义每小节的重音位置
- 视觉节拍指示：动画或闪烁提示
2.2.3 声音设置
- 多种节拍声音选择（木鱼、电子、鼓声等）
- 音量独立调节
- 重音与弱音区分
- 支持视觉静音（仅视觉提示，无声音）
2.2.4 练习辅助
- 节拍器倒计时功能
- 自动渐快/渐慢练习模式
- 静音练习模式（随机静音若干拍，训练内心节奏）

三、技术架构设计
3.1 技术选型
3.1.1 跨平台方案选择
推荐方案：React Native
- 优点：一套代码同时支持iOS和Android，开发效率高，社区活跃
- 音频处理：使用react-native-audio-toolkit或expo-av
- 性能：对于音频处理应用性能足够
备选方案：Flutter
- 优点：性能更好，UI更流畅
- 缺点：音频处理库相对较少
原生开发（iOS: Swift / Android: Kotlin）
- 优点：性能最佳，音频处理最精准
- 缺点：需要维护两套代码，开发周期长
建议：考虑到是个人项目且需要快速迭代，推荐使用React Native。
3.1.2 音频处理技术
- 音频采集：使用Web Audio API（如果是PWA）或原生音频模块
- 音高检测算法：YIN算法或自相关算法（Autocorrelation）
- FFT处理：使用DSP库进行快速傅里叶变换
- 滤波处理：带通滤波器，滤除吉他频率范围外的噪声（80Hz-400Hz）
3.1.3 状态管理
- React Native：使用Redux Toolkit或Zustand
- 本地存储：AsyncStorage保存用户设置
3.1.4 开发工具
- 代码编辑器：VS Code
- 版本控制：Git
- 调试工具：React Native Debugger、Flipper

4.2 音高检测模块（调音器核心）
4.2.1 算法选择：改进的YIN算法
YIN算法是目前最常用的音高检测算法之一，具有准确度高、计算复杂度适中的特点。
4.2.2 算法流程
1. 差分计算：计算信号的自相关函数
2. 累积均值归一化：增强峰值检测
3. 绝对阈值：寻找第一个低于阈值的谷值
4. 抛物线插值：提高频率估计精度
5. 频率转换：将周期转换为频率
4.2.3 吉他特定优化
- 频率范围限制：只检测80Hz-400Hz（吉他标准音域）
- 谐波处理：吉他的泛音丰富，需要处理谐波干扰
- 噪声门限：低于一定音量的输入视为噪声
- 稳定性检测：连续多帧检测结果一致才确认
4.2.4 性能指标
- 检测精度：±1音分
- 检测延迟：&lt; 50ms
- 信噪比要求：&gt; 20dB
- 最低可检测音量：约40dB SPL
4.2.5 核心接口
- processAudioData(buffer): 处理音频数据，返回检测到的频率
- setReferencePitch(hz): 设置参考音高（默认A4=440Hz）
- getNoteFromFrequency(freq): 将频率转换为音符名称
- getCentsDeviation(freq, note): 计算音分偏差

- 避免JavaScript setTimeout的不精确性

- 使用GainNode控制音量包络
- 不同节拍类型使用不同频率：

  * 其他强拍：800Hz，音量80%
  * 弱拍：600Hz，音量60%
4.3.4 视觉同步
- 使用requestAnimationFrame更新视觉指示器
- 视觉和听觉同步误差&lt; 16ms（一帧）
- 支持视觉节拍动画（如圆形缩放、颜色变化）
4.3.5 核心接口

6.1 音高检测算法伪代码
function detectPitch(audioBuffer):
    // 1. 预处理：加窗
    windowedBuffer = applyHannWindow(audioBuffer)
    
    // 2. 差分函数（Difference Function）
    for tau in range(1, maxPeriod):
        diff[tau] = 0
        for i in range(0, bufferSize - tau):
            diff[tau] += (windowedBuffer[i] - windowedBuffer[i + tau])^2
    
    // 3. 累积均值归一化（CMNDF）
    cmndf[0] = 1
    for tau in range(1, maxPeriod):
        sumDiff = sum(diff[1:tau+1])
        cmndf[tau] = diff[tau] / (sumDiff / tau) if sumDiff &gt; 0 else 1
    
    // 4. 寻找第一个低于阈值的谷值
    threshold = 0.1
    tauEstimate = null
    for tau in range(minPeriod, maxPeriod):
        if cmndf[tau] &lt; threshold:
            tauEstimate = tau
            break
    
    if tauEstimate is null:
        return null  // 未检测到有效音高
    
    // 5. 抛物线插值提高精度
    betterTau = parabolicInterpolation(cmndf, tauEstimate)
    
    // 6. 转换为频率
    frequency = sampleRate / betterTau
    
    return frequency
// 辅助函数：抛物线插值
function parabolicInterpolation(buffer, peakIndex):
    if peakIndex &lt;= 0 or peakIndex &gt;= len(buffer) - 1:
        return peakIndex
    
    alpha = buffer[peakIndex - 1]
    beta = buffer[peakIndex]
    gamma = buffer[peakIndex + 1]
    
    peakOffset = 0.5 * (alpha - gamma) / (alpha - 2*beta + gamma)
    return peakIndex + peakOffset
6.2 音符频率对照表
吉他标准调弦（E A D G B E）：
| 琴弦 | 音符 | 频率(Hz) | MIDI音符 |
|------|------|----------|----------|
| 6弦 | E2 | 82.41 | 40 |
| 5弦 | A2 | 110.00 | 45 |
| 4弦 | D3 | 146.83 | 50 |
| 3弦 | G3 | 196.00 | 55 |
| 2弦 | B3 | 246.94 | 59 |
| 1弦 | E4 | 329.63 | 64 |
音分计算公式：
cents = 1200 * log2(frequency / referenceFrequency)
其中：
- 1个八度 = 1200音分
- 1个半音 = 100音分
- 正值表示偏高，负值表示偏低
